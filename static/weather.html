⚙️ Ensemble Aggregation Pipeline
1. Fetch all sources → Parallel requests with circuit breakers
2. Z-score filter → Exclude models >3σ from ensemble mean
3. Dynamic weighting → EWMA of last 30 days per city
4. Weighted mean → TWC 1.5x base + dynamic adjustment
5. Bias correction → Per-city empirical bias from forecast_log
6. Empirical CDF → ECMWF ENS 51 members → rank-based probability
7. Isotonic calibration → PAVA algorithm, no parametric assumption
8. Kelly sizing → CV Kelly (1000 bootstrap, 10000 MC paths)
📊 Dynamic Source Weighting (EWMA)

Source weights are recalculated daily per city using Exponentially Weighted Moving Average of recent forecast errors (last 30 days).

  • Decay=0.15 → Recent errors weighted ~10x more than old ones
  • Range [0.3x, 2.5x] → Aggressive differentiation between good/bad sources
  • Per-city learning → Some sources are better for specific cities
  • Weather.com base 1.5x → Resolution source gets permanent boost
Self-improving: As more trades resolve, source weights automatically adjust. A source that outperforms for your city gets up to 2.5x weight.
📐 Bias Correction System

📈 Isotonic Calibration (PAVA)

Unlike Platt scaling (which assumes a sigmoid curve), isotonic regression learns the calibration function directly from data using the Pool Adjacent Violators Algorithm (PAVA).

  • No parametric assumption → Learns any shape
  • Monotonic constraint → Higher confidence = higher win rate
  • ECE (Expected Calibration Error) → Reported in dashboard
  • Auto-updates → Every time a trade resolves
Why isotonic? Real calibration curves aren't smooth sigmoids. They have flat regions, steep regions, and plateaus. Isotonic finds the true shape.
🔄 Walk-Forward Backtesting

Standard backtesting evaluates on ALL historical data at once. This is cheating — you're using future data to make past predictions.

Walk-forward splits time chronologically:

  • Train: Days 1-30 → compute per-source bias
  • Test: Day 31 → predict using ONLY days 1-30
  • Train: Days 1-31 → recompute bias
  • Test: Day 32 → predict using ONLY days 1-31
  • ... repeats through all data
This gives the TRUE expected accuracy. Not an overfitted backtest.

💰 Kelly Criterion & Position Sizing
  • CV Kelly Haircut: 1000 bootstrap iterations + 10,000 Monte Carlo equity paths
  • 4-phase scaling: SEED ($0-1K) → GROWTH ($1K-10K) → ACCELERATION ($10K-100K) → PRESERVATION ($100K+)
  • Max drawdown guard: 95th percentile DD ≤ 20%
  • Weather max bet: $25 per position (uncorrelated small bets)
🤖 Self-Improvement Cycle

Every resolved trade feeds back into the system:

  1. Market resolves → actual temperature recorded
  2. Source error computed → logged to source_city_rmse table
  3. EWMA recalculated → source weights shift automatically
  4. Bias updated → per-city correction refined
  5. Isotonic calibration retrained → PAVA on all historical predictions
  6. Walk-forward reruns → true accuracy metrics updated
The system gets smarter with every trade. No manual tuning required. All weights, biases, and calibrations are data-driven.
